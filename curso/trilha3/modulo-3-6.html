Módulo 3.6 Prático

Prática: agentificando um software real

Hands-on completo: da escolha do software-alvo à publicação do CLI, passando por setup, execução do pipeline, avaliação, refinamento e distribuição.

Avançado Nível
Prático Tipo
1

🎯 Escolhendo o alvo

O sucesso de CLI-Anything começa com a escolha correta do software-alvo. Critérios claros evitam frustração e maximizam a chance de gerar um CLI real e funcional.

Checklist de seleção

□ Código aberto: repositório público no GitHub/GitLab com licença permissiva
□ Backend separado da UI: diretórios distintos para API/lógica e interface gráfica
□ API interna identificável: REST endpoints, GraphQL, ou funções exportadas
□ Tamanho médio: 5K-50K LOC — nem trivial demais, nem massivo demais
□ Roda localmente: pode ser executado na sua máquina (Docker, npm start, etc.)
□ Operações repetitivas: CRUD, queries, exports — coisas que se beneficiam de CLI

✅ Bons candidatos

  • •App de notas com API REST (Trilium, Joplin server)
  • •Sistema de tarefas (Kanboard, Vikunja)
  • •Ferramenta de bookmarks (Linkding, Shiori)
  • •CMS headless (Strapi, Directus)
  • •Dashboard de monitoramento (Uptime Kuma)

❌ Evitar

  • •Jogos (toda lógica é visual/interativa)
  • •Editores gráficos puros (Excalidraw, tldraw)
  • •Apps SPA sem backend (toda lógica no browser)
  • •Software com toda lógica em stored procedures
  • •Projetos > 200K LOC (contexto excede limites)
2

🔧 Setup do ambiente

Antes de rodar o pipeline, você precisa de um ambiente preparado: agente CLI instalado, software-alvo clonado e funcionando localmente.

Setup passo a passo

1. Instalar pré-requisitos

# Node.js 18+ (necessário para maioria dos CLIs gerados)
node --version  # v18.x ou superior

# Agente CLI (escolha um)
npm install -g @anthropic-ai/claude-code  # Claude Code
npm install -g @anthropic-ai/gemini-cli   # Ou Gemini CLI

2. Clonar o software-alvo

# Exemplo: Linkding (bookmark manager)
git clone https://github.com/sissbruecker/linkding.git
cd linkding

3. Verificar que roda localmente

# Seguir instruções do README do projeto
docker compose up -d  # ou npm start, python manage.py runserver etc.

# Verificar que está acessível
curl http://localhost:9090/api/  # Deve retornar resposta

4. Configurar o agente

# Configurar API key
export ANTHROPIC_API_KEY=sk-ant-...

# Abrir o agente no diretório do projeto
claude  # ou gemini

💡 Dica de economia

O pipeline consome tokens significativos. Antes de rodar no projeto completo, faça um teste em um projeto pequeno (ex: um TODO app simples com < 1K LOC) para validar que seu ambiente está configurado corretamente. Isso evita gastar tokens em execuções que falham por problemas de setup.

3

📋 Executando o pipeline

Com o ambiente pronto, é hora de executar o pipeline CLI-Anything. O processo é orquestrado pelo agente e leva de 15 a 45 minutos dependendo da complexidade.

Passo a passo da execução

# Dentro do agente, no diretório do projeto:

> /cli-anything

# O agente inicia o pipeline automaticamente:
# [Fase 1/7] Analisando estrutura do projeto...
#   ✓ Linguagem: Python (Django)
#   ✓ Framework: Django REST Framework
#   ✓ Diretórios: bookmarks/, api/, templates/
#
# [Fase 2/7] Mapeando arquitetura...
#   ✓ 4 domínios identificados: bookmarks, tags, users, settings
#   ✓ 12 endpoints REST mapeados
#   ✓ 8 modelos de dados encontrados
#
# [Fase 3/7] Projetando CLI...
#   ✓ 18 comandos definidos
#   ✓ Árvore: linkding [bookmarks|tags|users|config] [action]
#
# [Fase 4/7] Implementando...
#   ✓ Gerando handlers, state manager, REPL
#   ✓ Integração via REST API (localhost:9090)
#
# [Fase 5/7] Gerando testes...
#   ✓ 42 testes gerados (28 unitários, 14 E2E)
#
# [Fase 6/7] Gerando documentação...
#   ✓ README.md, --help para cada comando
#
# [Fase 7/7] Gerando SKILL.md...
#   ✓ SKILL.md com 18 comandos documentados
#
# ✓ Pipeline completo! CLI gerada em ./linkding-cli/

📊 O que esperar em cada fase

Fases 1-2 (5-10 min): O agente lê muitos arquivos, faz perguntas sobre a estrutura, gera o architecture map
Fase 3 (3-5 min): Decisões de design — pode perguntar sobre preferências de nomenclatura
Fase 4 (10-20 min): A mais longa — geração de código real, muitas iterações internas
Fases 5-7 (5-10 min): Geração de testes, docs, SKILL.md — mais rápido pois é baseado no código já gerado
4

🔍 Avaliando o resultado

Após o pipeline completar, é hora de avaliar sistematicamente o resultado. Não confie cegamente — verifique cada aspecto do CLI gerado.

Checklist de avaliação

□ Cobertura: Os comandos gerados cobrem as funcionalidades principais do software?
□ Saída JSON: myapp bookmarks list --json retorna JSON válido com estrutura padronizada?
□ REPL: myapp repl funciona com auto-complete e histórico?
□ Estado: Auth persiste entre comandos? Contexto mantido?
□ Integração real: Crie algo via CLI e verifique se aparece na GUI do software original
□ Testes: npm test (ou equivalente) passa? Quantos falham?
□ SKILL.md: Está completo e reflete os comandos realmente implementados?

🚨 Red flags na avaliação

  • •CLI cria dados em arquivo JSON local em vez de chamar a API do software
  • •Testes passam mas estão testando mocks, não o software real
  • •SKILL.md lista comandos que não existem no código gerado
  • •Todos os comandos retornam dados estáticos (hardcoded)
5

🔄 Iterando e refinando

Quando o resultado não é bom (e na primeira vez raramente é perfeito), o refinamento iterativo é a chave. Feedback específico ao agente produz melhorias significativas.

Técnicas de refinamento

Feedback específico: 
> O comando bookmarks list está retornando dados mockados.
> A API real está em http://localhost:9090/api/bookmarks/
> O token de auth é passado via header "Authorization: Token xxx".
> Refaça a integração usando a API real.
Hints sobre a arquitetura: 
> O software usa Django REST Framework. Os viewsets estão em
> bookmarks/api/serializers.py e bookmarks/api/views.py.
> O modelo principal é Bookmark em bookmarks/models.py.
> Há filtros em bookmarks/api/filters.py.
Corrigir mapeamentos: 
> Faltam comandos para: tags management, bookmark sharing,
> bulk import/export. Adicione esses domínios.
> Também preciso do comando "bookmarks search --query TERM".

💡 Regra prática: 3 iterações

Iteração 1: Pipeline gera CLI básica, possivelmente com mocks. Iteração 2: Feedback corrige integração, adiciona comandos faltantes, fixa output JSON. Iteração 3: Polish — melhorar mensagens de erro, adicionar edge cases nos testes, refinar SKILL.md. Na maioria dos casos, 3 iterações produzem um CLI publicável.

6

🚀 Publicando e distribuindo

Com o CLI refinado e testado, é hora de publicar e distribuir — tornando o CLI acessível para humanos e agentes no ecossistema mais amplo.

Canais de distribuição

npm (Node.js CLIs)

npm publish — instalável via npm install -g myapp-cli

PyPI (Python CLIs)

pip install myapp-cli — via twine upload

GitHub Releases

Binários compilados para Linux, macOS, Windows

Docker Hub

docker run myapp-cli bookmarks list

📊 Integração com agentes

Claude Code: Colocar SKILL.md no projeto ou em ~/.claude/skills/ para ser reconhecido como tool
Gemini CLI: Registrar como extensão no diretório de extensions com manifest
MCP: Converter SKILL.md em tool definitions e publicar como MCP server
Docker MCP Catalog: Registrar no catálogo para descoberta automática por agentes

💡 CLI como produto

O conceito de "CLI como produto" vai além de distribuição técnica. Um CLI bem feito e bem documentado agrega valor ao software original: permite automação, integração com agentes, scripting avançado. Alguns mantenedores de projetos open-source adotam CLIs geradas por CLI-Anything como parte oficial do projeto, publicando junto com releases.

Resumo Final

✓ Escolher o alvo certo (backend separado, API identificável, 5-50K LOC) determina o sucesso do pipeline
✓ Setup: agente CLI instalado, software rodando localmente, API acessível — validar antes de rodar o pipeline
✓ O pipeline leva 15-45 min e gera CLI completa com handlers, REPL, testes, docs e SKILL.md
✓ Avaliar sempre: verificar integração real vs mock, cobertura de funcionalidades, qualidade do output JSON
✓ 3 iterações de refinamento com feedback específico produzem um CLI publicável na maioria dos casos
✓ Publicar via npm/PyPI/GitHub Releases e integrar com agentes via SKILL.md, MCP ou extensões
