🤖 O que é Claude Code

A CLI oficial da Anthropic para desenvolvimento assistido por IA. Não é um chatbot com syntax highlighting — é uma ferramenta agêntica completa que edita código, inspeciona repositórios, executa comandos e faz revisão com diffs.

Posicionamento

Claude Code é posicionado como ferramenta agêntica para desenvolvedores. As capacidades centrais:

Edição

Modificação cirúrgica de código

Inspeção

Leitura profunda do repo

Execução

Comandos no terminal

Revisão

Diffs e code review

Claude Code usa os modelos Claude da Anthropic (Sonnet, Opus) com janela de contexto longa. O diferencial: o agente não apenas responde perguntas sobre código — ele lê, entende, propõe mudanças, aplica e verifica. O loop é completo: da análise à execução, com o humano sempre no controle.

💡 Diferencial chave

A filosofia do Claude Code é "o humano sempre revisa". Cada mudança é proposta como diff antes de ser aplicada. Isso é fundamentalmente diferente de agentes que executam mudanças automaticamente — aqui o desenvolvedor mantém controle total sobre o que entra no código.

2

📂 Inspeção de repositórios

Claude Code lê o repositório inteiro — estrutura, dependências, padrões de código, configs. Entende o contexto do projeto para dar respostas melhores e mais precisas.

📊 O que o agente vê

Estrutura: árvore de diretórios, nomes de arquivos, organização do projeto
Dependências: package.json, requirements.txt, Cargo.toml, go.mod — sabe o que o projeto usa
Padrões: estilo de código, convenções de naming, arquitetura (MVC, microservices, etc.)
Histórico: git log, branches, commits recentes — entende o que mudou e por quê

O poder da inspeção profunda: quando você pede "adicione autenticação JWT neste projeto", o Claude Code não gera código genérico. Ele vê que o projeto usa Express com TypeScript, que já tem um middleware de logging, que os testes usam Jest — e gera código consistente com o que já existe. Suporta múltiplas linguagens e frameworks.

Ferramentas de inspeção

Read

Lê arquivos com controle de range

Glob

Busca arquivos por padrão

Grep

Busca conteúdo via regex

3

✏️ Edição e diff

Claude Code propõe mudanças como diffs. O princípio fundamental: o humano sempre revisa antes de aceitar. Duas ferramentas principais para modificar código.

✅ Edit tool

  • •Modificações cirúrgicas: troca trechos específicos de código via string matching
  • •Preserva contexto: não reescreve o arquivo inteiro, só o necessário
  • •Replace all: renomeação de variáveis e strings em todo o arquivo
  • •Segurança: falha se o trecho não for único, evitando edições ambíguas

📝 Write tool

  • •Arquivos novos: cria arquivos que não existem ainda
  • •Reescrita completa: para quando o arquivo precisa ser recriado do zero
  • •Requer leitura prévia: deve ler o arquivo antes de sobrescrever (safety check)
  • •Preferência: Edit é preferido; Write só quando necessário

O fluxo de edição: Claude Code lê o arquivo, analisa o contexto, propõe a mudança como diff visual, e o usuário aceita ou rejeita. Esse ciclo de proposta → revisão → aprovação é o que diferencia desenvolvimento agêntico de automação cega.

💡 Por que diffs importam

Diffs são a linguagem universal dos desenvolvedores. Ao propor mudanças como diffs, Claude Code fala a mesma língua que git, code review e CI/CD. O desenvolvedor vê exatamente o que muda, onde muda e por que muda — sem surpresas.

4

🔌 MCP e plugins

MCP (Model Context Protocol) é o backbone de extensibilidade do Claude Code. Além disso: slash commands, skills e hooks completam o ecossistema.

Camadas de extensibilidade

MCP Servers

Servidores que expõem ferramentas adicionais ao Claude Code. Configurados em settings JSON. Exemplos: acesso a banco de dados, APIs externas, serviços de deploy.

Slash commands

Comandos acionados por / no chat. Atalhos para workflows complexos: /commit, /review-pr, /deploy. Podem ser customizados por projeto.

Skills

Conjuntos de habilidades especializadas que o agente pode invocar. Cada skill tem contexto e ferramentas próprias.

Hooks

Callbacks pré e pós ações. Permitem automação: rodar linter após edição, executar testes após mudança, notificar após commit.

MCP: o padrão aberto

MCP é um protocolo aberto criado pela Anthropic que permite a qualquer ferramenta se conectar a qualquer agente. Um MCP server escrito para Claude Code também pode funcionar com Gemini CLI ou outros agentes que suportem o protocolo. Isso cria um ecossistema interoperável.

💾 Memória e contexto

Claude Code opera com múltiplas camadas de memória — desde a conversa atual até instruções persistentes entre sessões, passando pelo contexto do repositório.

📊 Camadas de memória

Sessão: conversa atual — todo o histórico de mensagens, resultados de ferramentas e decisões tomadas
Persistente: memória entre sessões via arquivos — o agente pode salvar e recuperar informações
CLAUDE.md: arquivo de instrução de projeto — convenções, regras, padrões que o agente deve seguir
Repositório: contexto do repo sempre disponível — o agente pode ler qualquer arquivo a qualquer momento

O arquivo CLAUDE.md é particularmente poderoso: funciona como um "briefing" que o agente lê no início de cada sessão. Você pode definir convenções de código, stack tecnológica, regras de negócio, e o agente vai seguir essas instruções em todas as interações. É a forma mais eficiente de "treinar" o agente para o seu projeto.

💡 Dica prática

Crie um CLAUDE.md na raiz do seu projeto com: linguagem/framework, convenções de naming, regras de commit, padrões de teste, e qualquer informação que um novo desenvolvedor precisaria saber. O agente vai usar isso como contexto em todas as interações.

🏢 Uso enterprise

Como empresas adotam Claude Code com controle, compliance e segurança — permissões granulares, auditoria e governança.

Pilares enterprise

Permissões granulares

Controle fino sobre quais ferramentas e diretórios o agente pode acessar. Configurável por projeto, time ou organização.

Auditoria

Log completo de todas as ações: quais arquivos foram lidos, quais comandos executados, quais mudanças propostas e aceitas.

Compliance

Políticas de dados: quais dados podem ser enviados ao modelo, retenção, residência. SOC 2 e certificações enterprise.

Governança

Administração centralizada: configuração de políticas, monitoramento de uso, controle de custos por time/projeto.

✅ Fazer

  • •Definir CLAUDE.md com regras de segurança do projeto
  • •Configurar permissões restritivas e relaxar conforme necessário
  • •Ativar auditoria para rastrear todas as ações do agente

⚠️ Evitar

  • •Dar acesso irrestrito ao filesystem em produção
  • •Ignorar logs de auditoria em ambientes regulados
  • •Compartilhar credenciais via contexto do agente

✓ Edit e Write tools para modificações cirúrgicas — o humano sempre revisa via diffs antes de aceitar
