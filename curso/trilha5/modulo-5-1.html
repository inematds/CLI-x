Projeto 1: Seu primeiro agente CLI

Passo a passo completo para instalar, configurar e usar um agente CLI pela primeira vez. Da instalacao ate a primeira edicao de codigo assistida por IA.

6 Topicos
45 Minutos
Iniciante Nivel
Hands-on Tipo
1

🚀 Escolhendo o agente

Existem tres agentes CLI principais no mercado. Cada um tem forcas diferentes e a escolha depende do seu contexto e necessidades.

Tabela comparativa

Agente Melhor para Setup Custo
Claude Code Codigo, refactoring, contexto longo npm + API key API paga
Gemini CLI Google ecosystem, pesquisa web npm + conta Google Gratis (cota)
Codex CLI Geracao rapida, multimodal npm + API key OpenAI API paga

💡 Recomendacao para iniciantes

Claude Code e a melhor opcao para comecar. O setup e simples (um comando npm + uma API key), a interface e intuitiva, e ele se destaca em tarefas de codigo — que e o caso de uso mais comum para um primeiro projeto. Se voce ja usa o ecossistema Google, Gemini CLI tambem e otima escolha.

2

📦 Instalacao passo a passo

Pre-requisitos minimos e comandos exatos para ter o agente rodando no seu terminal em menos de 5 minutos.

Pre-requisitos

Node.js 18+ — verifique com node --version
Terminal moderno — iTerm2, Windows Terminal, Alacritty, ou terminal padrao do Linux
npm ou yarn — gerenciador de pacotes Node.js

Opcao A: Claude Code

1. Instale globalmente:

npm install -g @anthropic-ai/claude-code

2. Configure a API key (obtenha em console.anthropic.com):

export ANTHROPIC_API_KEY="sk-ant-..."

3. Verifique a instalacao:

claude --version
# Output esperado: claude-code v1.x.x

Opcao B: Gemini CLI

1. Instale globalmente:

npm install -g @anthropic-ai/gemini-cli

2. Autentique com conta Google:

gemini auth login
# Abre o navegador para autenticacao OAuth

3. Verifique:

gemini --version
# Output esperado: gemini-cli v1.x.x
3

💬 Primeira conversa

Hora de interagir com o agente pela primeira vez. Vamos comecar com prompts simples e explorar o que ele consegue fazer.

Iniciando o agente

Abra o terminal e digite:

claude

O agente vai iniciar e mostrar um prompt interativo aguardando sua instrucao.

Primeiros prompts para testar

Comece com algo simples:

> Ola, me diga o que voce consegue fazer

Peca para listar arquivos:

> Liste os arquivos no diretorio atual e me diga o que cada um faz

Peca para explicar codigo:

> Leia o arquivo package.json e me explique as dependencias

📊 Dica importante

Prompts especificos > prompts genericos. Em vez de "me ajude com esse codigo", diga "adicione tratamento de erro na funcao processPayment do arquivo src/payment.ts". Quanto mais contexto voce der, melhor o resultado.

4

📂 Explorando um repositorio

O verdadeiro poder de um agente CLI aparece quando voce o usa dentro de um projeto real. O agente entende contexto.

Passo a passo

1. Navegue ate o projeto e inicie o agente:

cd meu-projeto && claude

2. Peca uma analise geral:

> Analise a estrutura deste projeto. Qual linguagem, framework e padroes estao sendo usados?

Exemplo de output

Projeto: API REST de e-commerce
Linguagem: TypeScript
Framework: Express.js + Prisma ORM
Estrutura:
  src/
    controllers/  -- 8 controllers (users, products, orders...)
    models/       -- schemas Prisma
    routes/       -- rotas Express
    middleware/   -- auth, validation, error handling
  tests/          -- Jest com 47 testes
Padroes: MVC, repository pattern, JWT auth
Observacoes: Falta tratamento de erro em 3 controllers

💡 Conceito: contexto e tudo

Quando voce abre o agente dentro de um diretorio, ele automaticamente tem acesso a toda a arvore de arquivos. Ele le, analisa e entende a estrutura do projeto. Isso e fundamentalmente diferente de copiar e colar codigo num chatbot web — o agente tem o contexto completo.

6

🔒 Configurando permissoes

Agentes CLI podem executar comandos no seu sistema. Configurar permissoes adequadas e essencial para seguranca.

Tres niveis de permissao

ASK Sempre pergunta antes de executar

O agente mostra o comando e espera sua aprovacao. Mais seguro, mais lento.

AUTO Executa automaticamente

O agente executa sem perguntar. Mais rapido, mas requer confianca.

DENY Bloqueia a acao

O agente nao pode executar essa categoria de comando. Maximo controle.

Configuracao recomendada para iniciantes

// .claude/settings.json
{
  "permissions": {
    "file_read":    "auto",     // seguro: apenas le
    "file_write":   "ask",      // review cada edicao
    "shell":        "ask",      // review cada comando
    "web":          "auto"      // seguro: apenas busca
  }
}

⚠️ Nunca auto-approve para:

rm -rf

Delecao de arquivos

git push

Push para remote

deploy

Deploy em producao
