Projeto 2: Criando um MCP server basico

Construa do zero um MCP server de notas pessoais com resources e tools. No final, qualquer agente CLI podera criar, listar, buscar e deletar suas notas.

🔌 O que vamos construir

Um MCP server de notas pessoais completo — o tipo de projeto que demonstra todos os conceitos fundamentais do protocolo.

Escopo do projeto

1 Resource

notes://list

Lista todas as notas em JSON

4 Tools (CRUD)

C create_note
R list_notes
R search_notes
D delete_note

💡 Resultado final

Ao terminar este projeto, qualquer agente CLI (Claude Code, Gemini CLI) podera gerenciar suas notas pessoais. Voce pedira "crie uma nota sobre MCP" e o agente usara o seu server para criar, listar e buscar notas — tudo via protocolo padrao.

2

🏗️ Setup do projeto

Vamos criar a estrutura do projeto do zero com TypeScript e o SDK oficial do MCP.

Criando o projeto

1. Inicialize o projeto:

mkdir mcp-notes && cd mcp-notes
npm init -y
npm install @modelcontextprotocol/sdk zod
npm install -D typescript @types/node tsx

2. Configure o TypeScript:

npx tsc --init --target es2022 --module nodenext \
  --moduleResolution nodenext --outDir dist --rootDir src

Estrutura de arquivos

mcp-notes/
  src/
    index.ts      -- ponto de entrada do server
    notes.ts      -- logica de armazenamento
  package.json
  tsconfig.json

Arquivo base src/notes.ts — armazenamento em memoria:

interface Note {
  id: string;
  title: string;
  content: string;
  tags: string[];
  createdAt: string;
}

const notes: Map<string, Note> = new Map();
let nextId = 1;

export function createNote(title: string, content: string, tags: string[]): Note {
  const note: Note = {
    id: String(nextId++),
    title, content, tags,
    createdAt: new Date().toISOString()
  };
  notes.set(note.id, note);
  return note;
}

export function listNotes(tag?: string): Note[] {
  const all = Array.from(notes.values());
  return tag ? all.filter(n => n.tags.includes(tag)) : all;
}

export function searchNotes(query: string): Note[] {
  return Array.from(notes.values()).filter(n =>
    n.title.includes(query) || n.content.includes(query)
  );
}

export function deleteNote(id: string): boolean {
  return notes.delete(id);
}
3

📋 Definindo resources

Resources sao dados que o agente pode ler — como endpoints GET de uma API. O agente "consulta" o resource para obter informacoes.

Resource: notes://list

import { McpServer } from "@modelcontextprotocol/sdk/server/mcp.js";
import { listNotes } from "./notes.js";

const server = new McpServer({
  name: "mcp-notes",
  version: "1.0.0"
});

// Resource: lista de notas
server.resource(
  "all-notes",           // nome interno
  "notes://list",         // URI
  { mimeType: "application/json" },
  async () => ({
    contents: [{
      uri: "notes://list",
      text: JSON.stringify(listNotes(), null, 2)
    }]
  })
);

📊 Anatomia de um resource

URI: identificador unico — o agente usa para "acessar" o dado (ex: notes://list)
Nome: nome legivel para display no agente
mimeType: formato do conteudo retornado (JSON, text, etc.)
Handler: funcao async que retorna o conteudo

🔧 Definindo tools

Tools sao acoes que o agente pode executar — como endpoints POST/PUT/DELETE. Cada tool tem nome, descricao, schema de input e handler.

Tool: create_note

import { z } from "zod";

server.tool(
  "create_note",
  "Cria uma nova nota com titulo, conteudo e tags",
  {
    title: z.string().describe("Titulo da nota"),
    content: z.string().describe("Conteudo da nota"),
    tags: z.array(z.string()).optional()
      .describe("Tags para categorizar")
  },
  async ({ title, content, tags }) => {
    const note = createNote(title, content, tags ?? []);
    return {
      content: [{ type: "text", text: JSON.stringify(note, null, 2) }]
    };
  }
);

Tool: search_notes

server.tool(
  "search_notes",
  "Busca notas por texto no titulo ou conteudo",
  {
    query: z.string().describe("Texto para buscar")
  },
  async ({ query }) => {
    const results = searchNotes(query);
    return {
      content: [{
        type: "text",
        text: results.length
          ? JSON.stringify(results, null, 2)
          : `Nenhuma nota encontrada para "${query}"`
      }]
    };
  }
);

Tool: delete_note

server.tool(
  "delete_note",
  "Deleta uma nota pelo ID",
  {
    id: z.string().describe("ID da nota para deletar")
  },
  async ({ id }) => {
    const deleted = deleteNote(id);
    return {
      content: [{
        type: "text",
        text: deleted
          ? `Nota ${id} deletada com sucesso`
          : `Nota ${id} nao encontrada`
      }]
    };
  }
);
5

🧪 Testando localmente

Hora de conectar o MCP server a um agente CLI e ver tudo funcionando na pratica.

Inicie o transport (stdio)

Adicione ao final de src/index.ts:

import { StdioServerTransport } from
  "@modelcontextprotocol/sdk/server/stdio.js";

const transport = new StdioServerTransport();
await server.connect(transport);

Configurar no Claude Code

// .claude/settings.json
{
  "mcpServers": {
    "notes": {
      "command": "npx",
      "args": ["tsx", "./mcp-notes/src/index.ts"]
    }
  }
}

Testando com prompts

> Crie uma nota sobre MCP com tag "estudo"
> Liste todas as minhas notas
> Busque notas com a palavra "MCP"
> Delete a nota com ID 1

✓ Verificacao

Se o agente conseguiu criar, listar, buscar e deletar notas sem erro, seu MCP server esta funcionando. O agente descobre as tools automaticamente via protocolo e as usa conforme necessario.

📦 Empacotando

Com o server funcionando, e hora de distribuir para outros usarem — via npm, Docker, ou registro MCP.

Publicar no npm

# Compile TypeScript
npx tsc

# Configure o package.json
npm pkg set bin.mcp-notes="dist/index.js"
npm pkg set type="module"

# Publique
npm publish

Dockerfile

FROM node:22-slim
WORKDIR /app
COPY package*.json ./
RUN npm ci --production
COPY dist/ dist/
ENTRYPOINT ["node", "dist/index.js"]

💡 Dica: SKILL.md

Adicione um arquivo SKILL.md na raiz do projeto descrevendo as tools e como usa-las. Agentes CLI leem esse arquivo automaticamente para descobrir como interagir com seu server. E o "README para agentes".
