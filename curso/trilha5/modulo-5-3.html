Projeto 3: Agentificando uma app com CLI-Anything

Transforme uma aplicacao existente em ferramenta de agente usando o pipeline automatizado CLI-Anything. De software convencional a interface agentica em menos de uma hora.

6 Topicos
60 Minutos
Avancado Nivel
Hands-on Tipo
1

🎯 O software-alvo

Vamos usar uma todo app open source como exemplo. Escolher bem o alvo e metade do sucesso — nem todo software e bom candidato.

Criterios de um bom candidato

Backend separado

Node.js, Python ou Go com API clara

API REST interna

Endpoints bem definidos para cada operacao

CRUD claro

Create, Read, Update, Delete identificaveis

Tamanho medio

~2000 linhas — suficiente para ser real, pequeno o bastante para entender

Nosso exemplo: Todo App

# Clone o projeto de exemplo
git clone https://github.com/example/todo-app.git
cd todo-app
npm install

# Estrutura:
src/
  server.ts       -- Express server
  routes/         -- POST/GET/PUT/DELETE /todos
  models/         -- Todo schema
  db.ts           -- SQLite storage

⚠️ Maus candidatos

Apps puramente frontend (SPA sem backend), apps com autenticacao complexa (OAuth multi-step), monolitos gigantes (+50k linhas), apps sem API definida (tudo via UI). Nesses casos, CLI-Anything tera dificuldade em gerar uma interface CLI util.

2

🔧 Instalando CLI-Anything

CLI-Anything e um plugin que roda dentro de um agente CLI. Vamos instalar e configurar antes de executar o pipeline.

Pre-requisitos

Agente CLI instalado (Claude Code ou Gemini CLI)
Node.js 18+ e npm
Git para clonar repositorios

Instalacao

# Clone o plugin CLI-Anything
git clone https://github.com/nicosaldaner/cli-anything.git

# Instale dependencias
cd cli-anything && npm install

# Verifique a instalacao
npm run cli-anything -- --help

📊 Como funciona

CLI-Anything e um plugin/comando que usa um agente CLI como "motor". Ele analisa seu software, identifica operacoes, gera uma interface CLI completa, testes e documentacao — tudo automaticamente. Voce so precisa apontar para o diretorio do projeto.

3

📋 Executando o pipeline

O pipeline CLI-Anything tem multiplas fases automatizadas. Vamos acompanhar cada uma passo a passo.

Passo a passo

1. Navegue ate o projeto-alvo:

cd todo-app

2. Abra o agente:

claude

3. Execute o pipeline:

> /cli-anything

Fases do pipeline

Fase 1 Analise — Le o codigo, identifica linguagem, framework, endpoints, modelos (~3 min)
Fase 2 Design — Define comandos CLI, flags, subcomandos baseados nas operacoes (~2 min)
Fase 3 Geracao — Escreve o codigo CLI, REPL interativo, handlers (~10 min)
Fase 4 Testes — Gera suite de testes automatizados para o CLI (~5 min)
Fase 5 Docs — Gera SKILL.md, README, help text (~3 min)

Tempo total estimado: 15-45 minutos dependendo do tamanho do projeto.

4

🔍 Analisando o output

Apos o pipeline, varios arquivos foram gerados. Vamos entender cada um e avaliar a qualidade.

Arquivos gerados

todo-app/
  cli/
    index.ts         -- ponto de entrada do CLI
    commands/        -- um arquivo por comando
      create.ts
      list.ts
      complete.ts
      delete.ts
    repl.ts          -- modo interativo
  tests/
    cli.test.ts      -- testes automatizados
  SKILL.md            -- descricao para agentes

Checklist de avaliacao

☐ Comandos cobrem todas as operacoes CRUD?
☐ Output JSON funciona com --json flag?
☐ REPL interativo mantem estado entre comandos?
☐ Integracao com o backend e real (nao mock)?
☐ SKILL.md descreve todos os comandos e exemplos?
5

🧪 Testando o CLI

Hora de validar tudo na pratica — testes automatizados e testes manuais para garantir que o CLI funciona de verdade.

Testes automatizados

npm test

# Output esperado:
PASS  tests/cli.test.ts
  ✓ creates a todo (45ms)
  ✓ lists all todos (23ms)
  ✓ completes a todo (31ms)
  ✓ deletes a todo (28ms)
  ✓ outputs valid JSON with --json flag (19ms)

Tests: 5 passed, 5 total

Testes manuais

todo create "Estudar MCP" --tag estudo
# ✓ Todo criado: ID 1 - "Estudar MCP" [estudo]
todo list --json
# [{"id": 1, "title": "Estudar MCP", "done": false, "tags": ["estudo"]}]
todo complete 1
# ✓ Todo 1 marcado como completo
todo delete 1
# ✓ Todo 1 deletado

📊 Teste o REPL interativo

todo repl
todo> create "Revisar PR"
✓ Todo criado: ID 2
todo> list
1. [ ] Estudar MCP [estudo]
2. [ ] Revisar PR
todo> exit
6

🤖 Conectando a um agente

O circulo se fecha: o CLI gerado vira ferramenta de agente. Software que antes so tinha UI agora e operavel por IA.

Registrar como ferramenta MCP

// .claude/settings.json
{
  "mcpServers": {
    "todo": {
      "command": "node",
      "args": ["./todo-app/cli/mcp-bridge.js"]
    }
  }
}

Testando via agente

> Liste minhas tarefas e marque as atrasadas como urgente

O agente:

1. Le o SKILL.md para descobrir comandos disponiveis

2. Executa todo list --json para obter tarefas

3. Identifica tarefas atrasadas pela data

4. Executa todo update [id] --tag urgente para cada uma

5. Reporta o resultado

💡 O conceito fundamental

CLI-Anything transforma qualquer software com backend em ferramenta de agente. O ciclo completo: software existente → analise automatica → CLI gerado → SKILL.md → agente descobre e opera. O software que antes exigia um humano clicando na UI agora e operavel por IA.

Resumo Final

✓ Escolher o software-alvo certo e critico — busque backend separado, API REST, CRUD claro, tamanho medio
✓ O pipeline CLI-Anything automatiza 5 fases: analise, design, geracao, testes e documentacao
✓ Sempre valide o output — testes automatizados + testes manuais + checklist de qualidade
✓ SKILL.md e a ponte entre o CLI gerado e os agentes — sem ele, a descoberta automatica nao funciona
✓ O circulo se fecha: software convencional vira ferramenta de agente operavel por IA
