Modulo 6.1 Analise Avancado

As tres camadas: agente, protocolo e ferramenta

Entenda a arquitetura do ecossistema CLI agent: quem faz o que, como as pecas se conectam e onde investir seu tempo para maximizar impacto.

6 Topicos
30 Minutos
Avancado Nivel
Estrategico Tipo
1

🏗️ Camada 1: Agentes terminais

Os agentes sao os operadores do ecossistema — o cerebro que entende intencao, planeja e executa. Competem em modelo, contexto e qualidade de edicao.

Os principais agentes

Agente Empresa Forca principal
Claude Code Anthropic Contexto longo, edicao precisa, MCP nativo
Gemini CLI Google Ecossistema Google, pesquisa web, gws
Codex CLI OpenAI Geracao rapida, multimodal, sandbox
Copilot CLI GitHub Integracao GitHub, Actions, repositorios

💡 Conceito-chave

O agente e quem orquestra. Ele recebe intencao em linguagem natural, decompoe em passos, chama ferramentas e compoe o resultado. A qualidade do agente depende do modelo subjacente, da janela de contexto e da capacidade de usar ferramentas de forma inteligente.

2

🔌 Camada 2: Protocolos de integracao

Protocolos sao a cola que conecta agentes a ferramentas. Sem protocolo padrao, cada ferramenta precisa de integracao custom.

MCP: o padrao emergente

Model Context Protocol (MCP) — protocolo aberto para conectar agentes a ferramentas externas
Docker MCP Gateway — containeriza MCP servers para isolamento e seguranca
Catalogos de extensoes — marketplaces de skills e MCP servers prontos para uso

Antes vs Depois do MCP

Sem protocolo padrao

Cada agente precisa de plugin especifico. N agentes x M ferramentas = N*M integracoes. Esforco explode exponencialmente.

Com MCP

Ferramenta implementa 1 MCP server. Qualquer agente compativel conecta. N + M integracoes. Esforco linear.

3

🔧 Camada 3: Ferramentas verticais

Ferramentas sao as maos especializadas do ecossistema. Cada vertical tem dominio profundo. O valor esta na especializacao.

Ferramentas por vertical

Workspace gws — Google Workspace operado por agente (Gmail, Drive, Sheets, Calendar)
Infraestrutura Pulumi — Infrastructure as Code com MCP server nativo
Cloud Azure Developer CLI — agentes hospedados em nuvem corporativa
Containers Docker — MCP Gateway para isolamento de ferramentas

📊 Dado importante

Ferramentas verticais tem dominio profundo que agentes generalistas nao tem. Um agente pode orquestrar, mas precisa de ferramentas especializadas para executar com precisao em cada vertical — Pulumi entende dependencias de infra, gws entende permissoes do Google, etc.

4

🔗 Como as camadas interagem

O agente descobre ferramentas via protocolo. A ferramenta expoe capacidades via schema. O protocolo garante compatibilidade.

Diagrama de interacao


  ┌─────────────────┐
  │   AGENTE CLI    │  Claude Code, Gemini CLI, Codex CLI
  │  (orquestrador) │  Entende intencao, planeja, executa
  └────────┬────────┘
           │
           │ MCP Protocol (JSON-RPC)
           │
  ┌────────┴────────┐
  │   PROTOCOLO     │  MCP, Docker Gateway
  │   (conexao)     │  Discovery, schemas, transport
  └────────┬────────┘
           │
           │ Tools / Resources / Prompts
           │
  ┌────────┴────────┐
  │  FERRAMENTAS    │  gws, Pulumi, kubectl, Snyk
  │ (especializadas)│  Dominio profundo, execucao real
  └─────────────────┘

Fluxo na pratica

1. Agente recebe instrucao: "provisione um banco PostgreSQL na AWS"
2. Agente consulta ferramentas disponiveis via MCP (discovery)
3. Descobre Pulumi MCP server com tool "provision_database"
4. Chama a tool com parametros corretos via protocolo MCP
5. Pulumi executa provisionamento real e retorna status
6. Agente interpreta resultado e reporta ao usuario

Grid do ecossistema

💡 Observacao

Note que nenhuma empresa domina as tres camadas sozinha. Isso e intencional — o ecossistema e composto. Google tem agente + ferramenta mas usa protocolo da Anthropic. Docker nao tem agente mas domina o layer de isolamento. Pulumi nao tem agente nem protocolo mas e referencia em IaC.

6

🎯 Onde investir tempo

Dependendo do seu perfil, a estrategia de investimento de tempo muda. O MCP e o investimento mais duravel porque e agnostico de agente.

Estrategia por perfil

Se voce e dev

Domine um agente CLI (Claude Code ou Gemini CLI) + aprenda a criar MCP servers. Isso cobre 80% dos casos de uso. Construa skills e extensoes para seu workflow.

Se voce e ops/infra

Foque em ferramentas verticais: Pulumi, kubectl, Terraform com MCP. Entenda Docker MCP Gateway para isolamento. O agente e meio, a ferramenta e fim.

Se voce e lider/gestor

Entenda as tres camadas e como compor. Nao precisa dominar cada ferramenta, mas precisa saber como elas se conectam para tomar decisoes de adocao e investimento.

🎯 Dica estrategica

MCP e o investimento mais duravel. Agentes vao mudar (novo modelo, novo provider), ferramentas vao evoluir (novas versoes, novos players). Mas o protocolo que conecta tudo tende a ser estavel — assim como HTTP conecta browsers a servers ha 30 anos. Aprenda MCP e voce se protege contra mudancas nas outras camadas.

Resumo Final

✓ O ecossistema CLI agent tem tres camadas: agentes (cerebro), protocolos (cola) e ferramentas (maos)
✓ MCP e o protocolo emergente que permite integracao linear (N+M) em vez de exponencial (N*M)
✓ Nenhuma empresa domina todas as camadas — o ecossistema e composavel por design
✓ A estrategia de investimento depende do perfil: dev foca em agente+MCP, ops em ferramentas, lider em composicao
✓ MCP e o investimento mais duravel — agnostico de agente, estavel como protocolo
